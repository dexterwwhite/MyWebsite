Portfolio

  • Nutritist

    Nutritist is a nutrition website that I and four other group members worked on together with the use of the Agile software development methodology. This web application was built with the use of HTML, CSS, JavaScript, Python Flask, and PostgreSQL. Additionally, our group utilized 3 different APIs from Edamam in order to add functionality to many of the aspects of our website. My main contributions were to the Tracking, Recipe Recommendation, and the Home pages. Additionally, a large portion of my work was on the integration of our database and APIs' input and output on to our user interface.

    View the Nutritist Github repo!

  • Highly Available Distributed Social Networking Service

    My highly available social networking service is a social networking service that is meant to never go down! Consisting of 3 server clusters (that each consist of 2 servers and a synchronizer) as well as a coordinator, this service is highly scalable, available, and synchronized. This project was built with C++ and with the use of gRPC. Additionally, it utilizes several different distributed systems concepts in order to maintain its availability and synchronization. This project was very challenging, yet very rewarding due to the complexity of the concepts used.

    View the highly available distributed SNS GitHub repo!

  • eyeTrain

    EyeTrain is an eye-tracking rhythm game that was made by 3 other students and I for the TAMUhack 2021 hackathon. This project was built with Python, Pygame, openCV, and dlib as well as the open source gaze_tracking library. My main contributions were to the Pygame interface, or more specifically the music/object interaction, collisions, and gameplay. And guess what? This project won us second place at the hackathon!

    View the eyeTrain devpost!

  • Content Streaming Service GUI

    This project was meant to mimic a content streaming service, and was built by my 3 partners and I using the Waterfall software development methodology. We built this project using Java, Java Swing, and a PostgreSQL database. Using a table that consisted of over 20000 rows of content as well as almost an equally large number of users, my team and I created algorithms that would mimic the algorithms of an actual content streaming service. One of my key additions was the implementation of the "Director's Choice" algorithm, which utilized several different queries and functions in order to make a recommendation to a user!

    View the content streaming service GUI's GitHub repo!

  • Calming Planner

    The Calming Planner is a dementia-accessible planner mobile application that I created along with 3 other partners. Our main goal for this application was making it as accessible as possible to those with dementia, as well as following general accessibility guidelines throughout. This mobile application was designed with React Native as well as utilized an SQLite database. My role in the group was mainly focused on the database features as well as initial setup.

    View Calming Planner's GitHub repo!
